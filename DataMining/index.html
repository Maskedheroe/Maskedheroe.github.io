    # 数据挖掘笔记

    # 基本认识

    数据挖掘是由一系列能够帮助人们从大量数据中获取知识的核心算法构成的。它是一门融合了数据库系统、统计学、机器学习和模式识别等领域的概念的交叉学科。

    # 数据挖掘解决了什么问题

    # 目前遇到了什么问题

    面对爆炸增长的数据，急需一个功能强大和通用的工具，以便从海量数据发现有价值的信息。而数据挖掘便是来解决该问题

    # 该如何解决？

    观点：数据挖掘是从大量数据中挖掘有趣模式和知识的过程，数据源包括数据库、数据仓库、web、其他信息存储库或动态地流入系统的数据

    简洁明了-即从数据中挖掘知识

    # 数据仓库和数据挖掘的OLAP技术

    # 什么是数据仓库

    首先，数据仓库是一个面向主题的、集成的、时变的、非易失的数据集合，支持管理决策制定。 与关系数据库、事物处理系统和文件系统是有区别的。

    # 面向主题的

    数据仓库只关注决策者的数据建模与分析，而不是日常操作和事务处理。 因此，数据仓库不会关注对于决策无用的数据，而是提供决策主题的简明视图

    # 集成的

    数据仓库是将多个异种数据源（如关系数据库、一般文件和联机事务）集成在一起。

    # 时变的

    数据存储从历史的角度（过去5-10年）提供信息。

    # 非易失的

    数据仓库总是物理地分离存放数据。

    # 为什么需要一个分离的数据仓库

    # 分离是为了提高两个系统的性能

    若在操作数据库中处理OLAP查询，可能会降低操作任务的性能

    # 分离是为了提高两个系统的性能

    并行机制不适用于OLAP，OLAP查询一般只需要对数据记录进行只读访问

    # 决策支持需要历史数据，而操作数据库一般不维护历史数据

    # 数据立方体技术

    # 立方体实体化

    计算冰山立方体的优势 1、不需要保存或者显示那些值低于阈值的单元格（冰山条件）
    2、高效的方法甚至可以避免计算不需要的中间单元
    3、避免爆发式增长

    # 立方体计算方法

    # 一般启发式

    可以从先前计算的聚合中计算聚合，而不是从基本事实表中计算。

  • 最小的孩子：从最小的、先前计算的长方体计算一个长方体。
  • 缓存结果：缓存一个长方体的结果，从中计算其他长方体以减少磁盘I/O
  • 摊销扫描：同时计算尽可能多的长方体以摊销磁盘读取。
  • 共享排序：当使用基本排序的方法时，跨多个长方体共享排序成本。
  • 共享分区：当使用基于哈希的算法时，在多个长方体之间共享分区成本。

    • # 多路阵列聚合(MOLAP)

    表现：

  • 基于数组的“自上而下”算法(从ABC到AB)
  • 使用多为块
  • 多维度同时聚合
  • 中间聚合值被重新用于计算祖先长方体
  • 不能做Apriori修剪；没有冰山优化

    • 特点：
  • 有效计算少量维度的完整立方体
  • 如果有大量维度，则应使用“自顶向下”计算和冰山立方计算方法

    • 步骤：
  • 将数组分成块(适合内存的小子多维数据集)
  • 压缩稀疏数组寻址:(chunk_id.offset)
  • 通过以最优化访问每个单元的次数的顺序访问立方体单元来“多路计算聚合”，并降低内存访问和存储成本。

    • # 使用壳碎片进行在线查询计算

    方法：给定物化片段立方体

  • 将查询分片，同壳碎片
  • 从片段立方体中获取每个片段对应的TID列表
  • 将每个片段的TID列表相交以构建实例化的基表
  • 使用带有任何立方算法的基表计算数据立方体

    • # 数据预处理

    为什么要预处理数据？ 是因为要将数据质量满足应用要求。数据质量有三个要素： 准确性、完整性和一致性。 实效性、可信性（反映有多少数据是用户信赖的）和可解释性（反映数据是否容易理解）同样影响数据质量。

    # 数据预处理的主要任务

    四个步骤：数据清理、数据集成、数据归约、数据变换

    # 数据清理

    现实世界的数据一般是不完整的、有噪声的和不一致的。

    # 缺失值

    当许多元组的一些属性没有值时。可使用最可能的值填充缺失值。比如可以使用回归、贝叶斯形式化的方法或决策树归纳确定该缺失值。 好的数据库和数据输入设计将有助于在第一现场把缺失值或错误的数量降到最低。

    # 噪声

    噪声是被测量变量的随机误差或者方差。数据光滑技术用于去掉噪声，例如有分箱、回归、离群点分析这些方法。

    # 数据集成

    数据挖掘经常需要合并来自多个数据存储的数据。

    # 实体识别问题

    考虑的问题是来自多个信息源的等价实体该如何才能匹配？比如一个数据库的customer_id与另一个cust_number指的是相同的属性。

    # 冗余和相关分析

    冗余是数据集成的另一个重要问题，一个属性可能由另一个属性导出，则这个属性可能是冗余的。属性或维(列)命名的不一致也会导致冗余。 可以使用相关分析检测是否有冗余
    1、标称数据的卡方相关检验
    对于标称数据，两个属性A和B之间的相关联系可以通过卡方检验发现。卡方统计检验假设A和B是独立的。如果拒绝该假设，则证明A和B是统计相关的。仔细学习3.1的例题
    2、数值数据的相关系数
    对于数值数据，可以计算A和B的相关系数，估计其相关度。-1<=r(A,B)<=+1，若r(A,B)大于0则A和B是正相关的。该值越大，相关性越强。 所以，一个较高的r(A,B)可以作为冗余被删除。

    # 分类问题

    # 逻辑回归

    逻辑回归是来解决二分类问题的方法(解决分类问题)
    逻辑回归 = 线性回归 + sigmoid函数
    回顾：1、线性回归是用一条直线来拟合自变量和因变量之间的关系
    2、sigmoid函数： y = 1 / 1 + e^-z
    逻辑回归： y = 1 / 1 + e^( -w * x + b )

    # 分类和回归的区别

  • 回归模型的输出是 **连续** 的
  • 分类模型的输出是 **离散** 的

    • c = -[ylna + (1-y)ln(1-a)] 损失函数：预测值与实际值相似程度的函数，损失函数越小，模型越好。
    可以用spark实现
